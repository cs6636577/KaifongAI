# เอกสารประกอบระบบ (Documentation) - Kaifong AI

ระบบ **Kaifong AI** เป็นแพลตฟอร์มเว็บแอปพลิเคชันสำหรับจัดการและแจ้งเรื่องร้องเรียน/เรื่องร้องทุกข์ ซึ่งพัฒนาขึ้นโดยใช้เทคโนโลยี **Next.js (App Router)**, **TypeScript**, และระบบจำลองฐานข้อมูลผ่าน Mock JSON ในระบบมีบทบาทหลัก 2 ฝั่ง ได้แก่ **ผู้ดูแลระบบ (Admin)** และ **ผู้บริหาร (Director)** 

---

## 1. โครงสร้างเมนูและหน้าเว็บ (Routing & Pages)

ระบบมีการแยกหน้าเว็บออกเป็น 2 ฝั่งอย่างชัดเจนผ่านระบบ Routing ของ Next.js ดังนี้:

### 1.1 ฝั่งผู้ดูแลระบบ (Admin) - `/app/admin`
* **`/admin/dashboard`**: หน้าแดชบอร์ดหลัก แสดงสถิติคำร้องเรียนรวม, เคสรอดำเนินการ, เคสเสร็จสิ้น, กราฟแสดงสถิติต่างๆ และตารางประวัติคำร้องล่าสุด
* **`/admin/member-approval`**: หน้าสำหรับอนุมัติสมาชิกใหม่ที่สมัครเข้ามาในระบบ
* **`/admin/permission-management`**: หน้าสำหรับกำหนดบทบาท (แอดมิน, เจ้าหน้าที่, ผู้ตรวจสอบ) และระงับ/เปิดสิทธิ์การใช้งานสมาชิกที่อนุมัติแล้ว
* **`/admin/problem-type`**: หน้าสำหรับตั้งค่าประเภทเรื่องร้องเรียน (หัวข้อปัญหา) สามารถเปิด/ปิดการใช้งานและแสดงจำนวนเคสที่เกี่ยวข้องกันได้
* **`/admin/manual`**: คู่มือการใช้งานระบบแบ่งตามประเภทผู้ใช้งาน:
  * `/admin/manual/user` - สำหรับประชาชนทั่วไป
  * `/admin/manual/staff` - สำหรับช่างเทคนิค
  * `/admin/manual/reporter` - สำหรับเจ้าหน้าที่

### 1.2 ฝั่งผู้บริหาร (Director) - `/app/director`
* **`/director/dashboard`**: แดชบอร์ดภาพรวมขององค์กรแสดงสถิติเรื่องร้องเรียนและความคืบหน้าของปัญหา
* **`/director/evaluate`**: หน้าสำหรับการประเมินและติดตามการดำเนินงานของเคสต่าง ๆ:
  * `/director/evaluate/table` - แสดงตารางรายการเคสทั้งหมดที่อยู่ระหว่างประเมิน
  * `/director/evaluate/detail/[id]` - แสดงรายละเอียดเชิงลึกของเคสที่เลือก พิกัดที่ตั้ง และมีส่วนแสดงไฟล์แนบ
* **`/director/manual`**: คู่มือการใช้งานระบบเช่นเดียวกับฝั่ง Admin

## 2. โครงสร้างข้อมูลและการจัดเตรียม API (API Routes)

การสื่อสารระหว่าง Client และฐานข้อมูลจำลอง (Mock JSON) จะผ่าน API Route ดังต่อไปนี้:

* **`/api/member-approval/summary`**: คำนวณสถิติคำขออนุมัติวันนี้, ยอด Pending, Rejected, Approved และความเร็วเฉลี่ยในการอนุมัติสมาชิก
* **`/api/member-approval/table`**: ดึงข้อมูลรายชื่อสมาชิกทั้งหมด
* **`/api/member-approval/update-table`**: บันทึกการอนุมัติหรือปฏิเสธคำขอสมาชิกใหม่ลงไฟล์ Mock JSON
* **`/api/permission-management/summary`**: คำนวณจำนวนพนักงานที่ได้รับสิทธิ์ในแต่ละกลุ่มบทบาท (แอดมิน, เจ้าหน้าที่, ผู้ตรวจสอบ)
* **`/api/permission-management/table`**: ดึงข้อมูลสมาชิกเฉพาะกลุ่มที่สถานะอนุมัติ (approved) เพื่อความสะดวกในการเปลี่ยนบทบาทสิทธิ์
* **`/api/problem-type/summary`**: สรุปสถิติจำนวนประเภทปัญหาทั้งหมด, ปัญหาที่เปิดใช้งาน และปิดใช้งาน
* **`/api/problem-type/table`**: สรุปประเภทเรื่องร้องเรียนทั้งหมด และผนวกจำนวนเคสร้องเรียนจริงที่เกิดขึ้นในหัวข้อนั้น ๆ
* **`/api/summary`**: รวบรวมข้อมูลเคส คำร้อง และบันทึกประวัติการเปลี่ยนสถานะ ส่งต่อให้ `lib/summaryDashboard` คำนวณสรุปสถิติตัวเลขของการ์ดบนหน้าแดชบอร์ดหลัก
* **`/api/table`**: ดึงข้อมูลเคสคำร้องเรียนทั้งหมดพร้อมคัดกรอง และจัดเรียงจากเวลาใหม่สุดไปหาเก่าสุด

## 3. รายละเอียด Components ทั้งหมด (Component Catalog)

3. **`AdminSidebar.tsx`**

   * **การใช้/สัมพันธ์:** ใช้คัดกรองข้อมูลตารางเคสในหน้าแดชบอร์ด

6. **`ComplainToolbar.tsx`**

   * **การใช้/สัมพันธ์:** อำนวยความสะดวกในการกรองข้อมูลตารางเคสบนแดชบอร์ด

   * **การใช้/สัมพันธ์:** แสดงตารางประวัติเคส หรือข้อมูลทั่วไปที่ไม่ซับซ้อนมาก

8. **`DataTableBase.tsx`**
   * **หน้าที่:** ตารางฐานรากของข้อมูลที่ต้องการการปรับแต่งหัวตารางหรือเนื้อหาแถวเฉพาะตัว
   * **การใช้/สัมพันธ์:** เป็นต้นแบบการสร้างหน้าจอตารางข้อมูลอื่น ๆ

    * **การใช้/สัมพันธ์:** ทำงานร่วมกับหน้าต่างหรือแถบการกรองเคส

### 3.2 โฟลเดอร์ `components/ui/Director/` (ส่วนสำหรับหน้าผู้บริหาร/ผู้อำนวยการ)
